# Contributing to Movie Sentiment Analysis

Thank you for your interest in contributing to this project! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/movie-sentiment-analysis.git`
3. Create a virtual environment and install dependencies:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

## Development Workflow

1. Create a new branch for your feature: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test your changes
4. Commit your changes: `git commit -m "Add your feature"`
5. Push to your fork: `git push origin feature/your-feature-name`
6. Open a Pull Request on GitHub

## Code Style

- Follow PEP 8 style guide
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and small
- Add comments for complex logic

## Testing

Please ensure your code works correctly before submitting a PR. Test with:
- Different dataset sizes
- Various model configurations
- Edge cases

## Pull Request Process

1. Update the README.md if needed
2. Add or update tests if applicable
3. Ensure all tests pass
4. Request review from maintainers

## Reporting Issues

When reporting issues, please include:
- Description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Python version and OS

## Questions?

Feel free to open an issue for questions or discussions!
